SECTION 03 / METHOD

引擎三段式
I/O · 判定 · 聚合

采样（I/O）调用中转站，单轮 60 秒超时仅标记不中断； 401/403 立即中止任务。判定七个维度均为纯函数，便于属性测试。 聚合加权评分（inconclusive 维度剔除）；来源冲突时硬封顶总分， 派生身份替换 / 降智警示。

STAGE 01

采样

undici 直连中转站接口，提取响应协议元数据：id 前缀 / usage 字段 / 结束原因 / 流式帧序列。

→
STAGE 02

判定

七个纯函数判定器，每维输出 verdict（pass / warn / fail / inconclusive）+ 证据片段。

→
STAGE 03

聚合

按权重加权得出 0–100 总分；触发 identity_swap / downgrade 警示时硬封顶。

方法学参考论文《Auditing Black-Box LLM APIs with a Rank-Based Uniformity Test》、 《Are You Getting What You Pay For? Auditing Model Substitution in LLM APIs》等。 完整白皮书见 检测原理。

SECTION 06 / FAQ

与科普

Q·01

什么是 API「中转站」？

第三方在官方 API 之上做封装的服务商，对外提供 OpenAI / Claude / Gemini 兼容协议接口。 常见动机：聚合多家、降低单价、规避地域限制。风险：身份替换、降智、协议不一致、隐私不当处理。

Q·02

检测准确率有多高？

本检测是低成本技术性验证，不是法律审计。我们公开方法学并维护开源代码。 七维体系中，「协议来源指纹」是最难伪造、权重最高的维度；通过该维度的渠道更可信。 但不保证 100% 准确。

Q·03

提交的 API Key 安全吗？

HTTPS 传输 · 仅内存存活 · 任务结束 finally 即焚（参见 keyHolder.ts） · 日志掩码（首尾各 4 字符） · 历史仅存脱敏结果。我们不保存你的 Key。

Q·04

「降智」是怎么判定的？

多轮事实性问答 + 协议元数据比对，与该模型官方分布的统计偏差。 单次低分不会判降智，需多轮一致偏离才会触发警示。详见 检测原理白皮书 §4.2。

Q·05

榜单数据怎么更新？

渠道信息由运营管理维护；价格变更自动入库，用于计算近 7 天百分比。 可用率、降智、延迟以最近一次有效检测为准。欢迎用户上报新发现。
